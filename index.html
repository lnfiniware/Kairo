KairoDB v0.4.0 Stable

Human-readable databases.

A minimal database layer translating plain-text schemas into robust SQLite or PostgreSQL storage. Fast Rust engine. No complex ORM layers. Fully local-first.

Download v0.4.0 Source Code

Read Any Database

Instantly parse and translate any SQLite file or PostgreSQL schema into a clear, formatted human-readable representation.

$ kairo read data/kairo.db

Plain-Text Schemas

Define database structures without SQL clutter or heavy configs. Simply declare tables and columns inside minimal `.kairo` files.

$ kairo create users

Natural CLI Queries

Inspect records rapidly from your shell using simplified, conversational query strings or fallback to robust raw SQL expressions.

$ kairo query "from users"

Schema Compilation & Migration

Declare once, compile anywhere. KairoDB compiles declarative text files into optimized, standard schemas.

schema/users.kairo 
table users {
  name: string
  age: int
  active: bool = true
}

table posts {
  title: string
  body: string
  user_id: int [required]
}
kairo status 
# Migrating schema to data/kairo.db
✔ Schema successfully applied to SQLite!

kairo tables
----------------------------------------
- users (3 columns, 12 rows)
- posts (3 columns, 45 rows)

kairo read data/kairo.db
----------------------------------------
table users {
  id: int
  name: string
  age: int
  active: bool
}
Engine v0.4.0 Feature

Seamless Multi-Database Connectivity

KairoDB integrates natively with both standard **SQLite files** and live **PostgreSQL instances**. Point the reader at a raw file or supply a database server connection string—Kairo translates rows and metadata into clean, structured schemas on the fly.

SQLite Engine Local-first, direct file inspection
PostgreSQL Encrypted TLS strings, custom port mappings

AI & LLM Optimized Context

Engineered for AI Code Assistants & LLMs

Typical SQL definitions, migrations, and heavy ORM codebases bloat prompt context windows, causing coding assistants (such as Claude, ChatGPT, and local models) to lose accuracy and hallucinate imports. KairoDB is meticulously structured for AI agent workspaces.

Compact Token Overhead

Plain-text Kairo schemas use up to 90% fewer tokens than standard migrations, leaving more memory for core application instructions.

100% Deterministic CodeGen

Kairo's zero-boilerplate declarative syntax ensures that LLMs generate correct, bug-free schema configurations without requiring additional TypeScript or SQL typings.

Agent-Native Migrations

Automated shell executions like `kairo create` and human-readable DB exports let AI agents interact with databases seamlessly inside autonomous development loops.

Engineered for Minimalists

Ditch bloated frameworks. Manage data with high-performance local structures.

Zero Compilation Bloat

No heavy docker databases needed. Create, seed, and inspect complete schemas instantly via SQLite or remote PG endpoints.

Portable Global Tool

Written in native, optimized Rust. Ships as a single standalone executable. Runs instantly across Windows, macOS, and Linux without Rust runtimes.

Intuitive Local-First Flows

Human-readable syntax lets you inspect structures and rows as plain English objects, eliminating heavy external GUI dashboards.
